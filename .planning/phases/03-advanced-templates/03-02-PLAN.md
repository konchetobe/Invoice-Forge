  <name>Task 1: Extend PdfService with render mode and template context</name>
  <files>src/Services/PdfService.php</files>
  <action>
    **Extend `generate()` signature** — Add a fourth parameter `string $render_mode = 'pdf'` (values: 'pdf' or 'email'). Pass `$render_mode` through to the template include. When `$render_mode === 'email'`, skip mPDF entirely — just return the HTML string from ob_get_clean() directly (email body is plain HTML, not a PDF).

    **Create `getTemplateContext()` method** — New private method `getTemplateContext(int $invoice_id, string $render_mode): array`. Returns an associative array combining:
    - All existing `getInvoiceData()` fields
    - `render_mode` ('pdf' or 'email')
    - Template config from `invoiceforge_settings['template']`: `accent_color`, `logo_path`, `logo_url`, `id_no_label`, `section_order`, `section_visibility`, `signature_fields`
    - Company profile: `company_id_no`, `company_office`, `company_att_to`, `company_bank_name`, `company_iban`, `company_bic` (from invoiceforge_settings)
    - Client extended fields: `client_id_no`, `client_office`, `client_att_to` (from client post meta using the invoice's `_invoice_client_id`)
    - `payment_method` from `_invoice_payment_method` post meta
    - `has_discount` — boolean, true if any line item has non-zero discount_value
    - `currency_symbol` — derived from currency code (simple mapping: USD=>$, EUR=>euro, GBP=>pound, BGN=>лв, fallback to code)

    **Extend `getInvoiceData()`** — Add `payment_method` field from `_invoice_payment_method` post meta. Add client extended fields. Ensure line_items now include `discount_type` and `discount_value` (already handled if Plan 01 updated LineItem.toArray()).

    **Create `renderEmailBody()` method** — Public method `renderEmailBody(int $invoice_id): string`. Calls `generate($invoice_id, 'S', '', 'email')` and returns the HTML string. This is the method EmailService will call in Plan 03.

    **Update template include** — In `generate()`, replace the bare `include` with:
    ```php
    $template_vars = $this->getTemplateContext($invoice_id, $render_mode);
    ob_start();
    extract($template_vars, EXTR_SKIP);
    include $template_file;
    $html = ob_get_clean();
    ```

    When `$render_mode === 'email'`, return `$html` directly without creating mPDF instance. When `$render_mode === 'pdf'`, proceed with mPDF as before.

    **mPDF config** — Add `table-layout: fixed` default CSS to the mPDF constructor config via the `default_font` and `defaultCssFile` or inline `$mpdf->WriteHTML('<style>table{table-layout:fixed;}</style>', \Mpdf\HTMLParserMode::HEADER_CSS)` before writing the main HTML.

<task type="auto">
  <name>Task 2: Create configurable PDF/email invoice template</name>
  <files>templates/pdf/invoice-default.php</files>
  <action>
    Create `templates/pdf/invoice-default.php` — the single configurable invoice template. This file is `include`d by PdfService with all template variables available via `extract()`.

    **CRITICAL mPDF rules:**
    - ALL multi-column layouts use `<table>` — NO CSS flexbox or grid
    - Block elements (`<div>`, `<p>`) inside `<td>` lose block behavior — use `<br>` for line breaks within cells
    - Use `style="table-layout:fixed;"` on tables
    - Logo: use absolute server path (`$logo_path`) for PDF mode, public URL (`$logo_url`) for email mode
    - All widths as percentages summing to 100%

    **Template structure** — The template branches on `$render_mode`:

    ### PDF Mode (`$render_mode === 'pdf'`):

    Render sections in the order specified by `$section_order` array. Use a loop over `$section_order` with a switch/match to render each section. Check `$section_visibility` before rendering optional sections.

    **Section: header** (always visible)
    Three-column table (38% | 24% | 38%):
    - Left cell (BUYER): Bold "BUYER" header, client company/name, address, `$id_no_label`: `$client_id_no` (if set), Office: `$client_office` (if set), Att To: `$client_att_to` (if set), VAT No if set.
    - Center cell: Accent-colored background, white text. "INVOICE" label, invoice number, date, due date.
    - Right cell (SELLER): Logo image (if `$logo_path` set and file_exists, `<img src="$logo_path" style="max-height:60px; max-width:180px;">`), company name, address, `$id_no_label`: `$company_id_no`, Office: `$company_office`, Att To: `$company_att_to`, company phone, company email.

    **Section: line_items** (always visible)
    Table with columns: No., Description, Qty, Unit Price, [Discount %, Discount Value — only if `$has_discount`], Tax %, Tax Amount, Total.
    - Column headers with accent color background, white text.
    - Each row from `$line_items` array.
    - Row numbering starts at 1.

    **Section: totals** (always visible)
    Right-aligned totals block (use a table with empty left cell and right cell):
    - Subtotal
    - Document discount row (if `$section_visibility['discount_row']` AND `$discount_value > 0`): show discount type and value
    - Tax total
    - **Total** (bold, accent-colored background)

    **Section: bank** (conditional — shown when `$payment_method === 'Bank transfer'`)
    Two-column table:
    - Left: Payment method label, Bank name (`$company_bank_name`), IBAN (`$company_iban`), BIC (`$company_bic`)
    - Right: (empty or additional info)

    **Section: notes** (if `$section_visibility['notes']` is true AND `$notes` is not empty)
    Full-width block below footer: "Notes" header, then `$notes` content.

    **Section: signature** (if `$section_visibility['signature']` is true)
    Two-column table from `$signature_fields`:
    - Left column fields (where `col === 'left'`): label + underline
    - Right column fields (where `col === 'right'`): label + underline

    **VAT note** — Auto-shown below totals when invoice has zero tax (VAT-exempt text) or when a reverse charge flag is set. Use conditional: `if ($tax_total == 0) { echo VAT exempt note }`.

    **Payment instructions** — If `$payment_instructions` is not empty, show below bank section.

    **Styling:**
    - Base: black text on white, professional font (DejaVu Sans for UTF-8/Cyrillic support in mPDF)
    - Accent color (`$accent_color`) validated with regex fallback to `#1a2b4a`
    - Bold section headers
    - Simple 1px solid borders on tables
    - All text uses `esc_html()` for output escaping
    - All labels wrapped in `__('Label', 'invoiceforge')` for i18n

    ### Email Mode (`$render_mode === 'email'`):

    Simplified HTML body (NOT sent through mPDF). Uses inline styles only (email client compat). Uses only `<table>`, `<tr>`, `<td>`, `<p>`, `<strong>`, `<a>` elements.

    Content:
    - Company name header
    - "Invoice {number}" heading
    - Summary table: Date, Due Date, Amount Due, Status
    - Line items summary table (description, qty, total — no discount/tax columns)
    - Total amount (bold)
    - "Pay Invoice" button — `<a>` styled as button with `href="#"` and `data-invoice-id` attribute (placeholder for Phase 8 payment gateway link). Text: "Pay Invoice". Style: accent-colored background, white text, padding, border-radius.
    - Footer: company name, address, email
    - Logo: use `$logo_url` (public URL) as `<img>` if set

    All email HTML uses inline `style=""` attributes. No `<style>` blocks, no `class` attributes (stripped by email clients).
  </action>
